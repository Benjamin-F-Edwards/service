Publish to GitHub Pages — free public URL

Follow these steps once. After that, just replace index.html each week to update the live dashboard for everyone.

  1. Create a free GitHub account if you don't have one.
    github.com/signup →
  2. Create a new repository. Go to github.com/new, name it something like service-dashboard and set visibility to Public. Click Create repository.
  3. Download your dashboard as an HTML file. Click the ··· menu on this artifact → Download (or copy the code and save it as a file named exactly): index.html
  4. Upload the file to your repository. On your new repo page, click Add file → Upload files, drag in index.html, and click Commit changes.
  5. Enable GitHub Pages. Go to your repo → Settings → Pages. Under Source, select Deploy from a branch, choose main branch and / (root) folder. Click Save.
  6. Your dashboard is live! After ~60 seconds, your public URL will be: https://YOUR-USERNAME.github.io/service-dashboard/
    💡 Replace YOUR-USERNAME with your actual GitHub username. Share this link with anyone — no login needed.
  7. Updating each week is easy: paste new data into Claude, download the refreshed artifact, go back to your repo → Add file → Upload files, upload the new index.html (it will replace the old one), and commit. The URL stays the same — your audience just refreshes the page.
🔒 Want to keep it private? GitHub Pages requires a public repo on free accounts. For a private dashboard, consider Netlify (free tier, password-protect with a paid plan) or host on your company's intranet.
